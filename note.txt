Pythonにおける一般的な書き方と高速化された書き方の実行時間を比較し、計算量（O(N) や O(1) など）の違いによる速度差を実証するコードを作成する。

【全体要件（必須事項）】

計測モジュール: 必ず標準ライブラリの timeit を使用すること（time.time() は非推奨）。

実行回数: 各テストにつき、number=100（100回実行）を repeat=5（5セット）行い、その中の最小値を採用すること。

データサイズ（N）: 要素数 N を 10,000, 50,000, 100,000, 500,000 の4段階で変動させ、スケールした際の速度変化を計測すること。

出力形式: 最終的な結果は、そのまま記事に貼り付けられるMarkdown形式のテーブル、および後でグラフ化しやすいようにCSV形式の2パターンで標準出力（print）すること。

【テストケース詳細】

以下の4つのカテゴリについて、それぞれの実行時間を比較する関数を実装してください。

テスト1：先頭への要素追加（リスト vs deque）

準備: 空の list と collections.deque を用意する。

比較対象:

list に対し、insert(0, i) を N 回繰り返す処理。

deque に対し、appendleft(i) を N 回繰り返す処理。

検証ポイント: リストの先頭追加が O(N) となり、データ増大に伴い致命的に遅くなることを示す。

テスト2：要素の存在確認（リスト vs セット）

準備: 1 から N までの連番を格納した list と set をそれぞれ用意する。

比較対象:

list に対し、N - 1 （末尾近くの値）が in 演算子で存在するか確認する処理。

set に対し、N - 1 が in 演算子で存在するか確認する処理。

検証ポイント: セットの検索が O(1) であり、リストの線形探索 O(N) との圧倒的な差を示す。

テスト3：ループ処理によるリスト生成（for文 vs 内包表記 vs map）

準備: 1 から N までの連番リスト（base_data）を用意する。

比較対象:

空リストを用意し、for ループで回して append(i * 2) する処理。

リスト内包表記 [i * 2 for i in base_data] を使う処理。

map 関数を使い list(map(lambda x: x * 2, base_data)) とする処理。（※別途、事前定義した通常の関数を適用するバージョンも余裕があれば追加）

検証ポイント: Pythonインタプリタのループオーバーヘッドの違いを示す。

テスト4：文字列の結合（+= vs join）

準備: 長さ10のランダムな文字列を N 個格納したリストを用意する。

比較対象:

空文字列 "" に対し、for ループで += を使って文字列を結合していく処理。

"".join() を使ってリスト内の文字列を一括結合する処理。

検証ポイント: メモリの再確保によるオーバーヘッドの違いを示す。

【コードの構造設計】

各テストケースを実行する独立した関数を作成する。

N の配列をループで回し、各テスト関数を呼び出すメイン関数を作成する。

不要なガベージコレクションによる計測ブレを防ぐため、timeit の実行中は gc.disable() を行い、終わったら gc.enable() を行うこと。
